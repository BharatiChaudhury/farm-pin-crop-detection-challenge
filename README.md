I used this pipeline to enter Zindi’s <a href="https://zindi.africa/competitions/farm-pin-crop-detection-challenge" data-href="https://zindi.africa/competitions/farm-pin-crop-detection-challenge" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Farm Pin Crop Detection Challenge</a>. I may not have won the contest but I learnt some great techniques for working with remote-sensing data which I detail in this post.</p><p name="e61c" id="e61c" class="graf graf--p graf-after--p">Here are the preprocessing steps I followed:</p><ol class="postList"><li name="497e" id="497e" class="graf graf--li graf-after--p">divided an area of interest into a grid of ‘patches’,</li><li name="f552" id="f552" class="graf graf--li graf-after--li">loaded imagery from disk,</li><li name="cfdc" id="cfdc" class="graf graf--li graf-after--li">masked out cloud cover,</li><li name="0ccb" id="0ccb" class="graf graf--li graf-after--li">added NDVI and euclidean norm features,</li><li name="d5a1" id="d5a1" class="graf graf--li graf-after--li">resampled the imagery to regular time intervals,</li><li name="3e27" id="3e27" class="graf graf--li graf-after--li">added raster layers with the targets and identifiers.</li></ol><p name="0de2" id="0de2" class="graf graf--p graf-after--li">I reframed the problem of crop type classification as a semantic segmentation task and trained a U-Net with a ResNet50 encoder on multi-temporal multi-spectral data using image augmentation and mixup to prevent over-fitting.</p><p name="6aa2" id="6aa2" class="graf graf--p graf-after--p">My solution borrows heavily from the approach outlined by <a href="https://medium.com/u/fe489d29ae67" data-href="https://medium.com/u/fe489d29ae67" data-anchor-type="2" data-user-id="fe489d29ae67" data-action-value="fe489d29ae67" data-action="show-user-card" data-action-type="hover" class="markup--user markup--p-user" target="_blank">Matic Lubej</a> in his <a href="https://medium.com/sentinel-hub/land-cover-classification-with-eo-learn-part-1-2471e8098195" data-href="https://medium.com/sentinel-hub/land-cover-classification-with-eo-learn-part-1-2471e8098195" class="markup--anchor markup--p-anchor" target="_blank">three</a> <a data-href="https://medium.com/sentinel-hub/land-cover-classification-with-eo-learn-part-2-bd9aa86f8500" class="markup--anchor markup--p-anchor" target="_blank">excellent</a> <a data-href="https://medium.com/sentinel-hub/land-cover-classification-with-eo-learn-part-3-c62ed9ecd405" class="markup--anchor markup--p-anchor" target="_blank">posts</a> on land cover classification with <a href="https://github.com/sentinel-hub/eo-learn" data-href="https://github.com/sentinel-hub/eo-learn" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">eo-learn</a>.</p><p name="c648" id="c648" class="graf graf--p graf-after--p">The python notebooks I created can be found in this github repository: <a href="https://github.com/simongrest/farm-pin-crop-detection-challenge" data-href="https://github.com/simongrest/farm-pin-crop-detection-challenge" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">https://github.com/simongrest/farm-pin-crop-detection-challenge</a></p><h3 name="786b" id="786b" class="graf graf--h3 graf-after--p">The challenge</h3><p name="4afb" id="4afb" class="graf graf--p graf-after--h3"><a href="https://zindi.africa/" data-href="https://zindi.africa/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Zindi</a> is an African competitive data science platform that focusses on using data science for social benefit. In Zindi’s 2019 <a href="https://zindi.africa/competitions/farm-pin-crop-detection-challenge" data-href="https://zindi.africa/competitions/farm-pin-crop-detection-challenge" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Farm Pin Crop Detection Challenge</a>, participants to <strong class="markup--strong markup--p-strong">trained machine learning models using </strong><a href="https://sentinel.esa.int/web/sentinel/missions/sentinel-2" data-href="https://sentinel.esa.int/web/sentinel/missions/sentinel-2" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><strong class="markup--strong markup--p-strong">Sentinel2</strong></a><strong class="markup--strong markup--p-strong"> imagery in order to classify the crops being grown in fields </strong>along a stretch of the Orange River in South Africa.</p><p name="229e" id="229e" class="graf graf--p graf-after--p">The data supplied to contestants consisted of two shape files containing the training set and test set’s field boundaries, as well as Sentinel2 imagery for the area of interest at 11 different points in time between January and August 2017.</p></div><div class="section-inner sectionLayout--outsetColumn"><figure name="8928" id="8928" class="graf graf--figure graf--layoutOutsetCenter graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 1032px; max-height: 871px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 84.39999999999999%;"></div><img class="graf-image" data-image-id="1*ncEqYnxT6Yq9MdWp7sdKcA.png" data-width="5601" data-height="4727" src="https://cdn-images-1.medium.com/max/1200/1*ncEqYnxT6Yq9MdWp7sdKcA.png"></div><figcaption class="imageCaption">The section of the Orange River — grey test set fields interspersed amongst green training set fields</figcaption></figure></div><div class="section-inner sectionLayout--insetColumn"><p name="7241" id="7241" class="graf graf--p graf-after--figure">The training and test sets consisted of 2497 fields and 1074 fields respectively. Each field in the training set was labelled with one of nine labels indicating the crop that was grown in that field during 2017.</p><p name="fe62" id="fe62" class="graf graf--p graf-after--p">The crop types were:</p><pre name="17c2" id="17c2" class="graf graf--pre graf-after--p">Cotton<br>Dates<br>Grass<br>Lucern<br>Maize<br>Pecan<br>Vacant<br>Vineyard<br>Vineyard &amp; Pecan (“Intercrop”)</pre><p name="b477" id="b477" class="graf graf--p graf-after--pre">Competitors were only to use the provided data and (due to a data leak discovered during the competition) were prohibited from using Field_Id as a training feature.</p><h3 name="898d" id="898d" class="graf graf--h3 graf-after--p">Data pre-processing with eo-learn (<a href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Process%20Satellite%20Imagery.ipynb" data-href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Process%20Satellite%20Imagery.ipynb" class="markup--anchor markup--h3-anchor" rel="noopener" target="_blank">notebook</a>)</h3><p name="faa8" id="faa8" class="graf graf--p graf-after--h3">The eo-learn library allows users to divide up an area of interest into patches, define a workflow and then execute the workflow on the patches in parallel.</p><h4 name="337c" id="337c" class="graf graf--h4 graf-after--p">1. Splitting the area of interest</h4><p name="38eb" id="38eb" class="graf graf--p graf-after--h4">Using <code class="markup--code markup--p-code">BBoxSplitter</code> from the <code class="markup--code markup--p-code">sentinelhub</code> library I split the river up into 12 patches:</p><figure name="85e0" id="85e0" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 601px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 85.9%;"></div><img class="graf-image" data-image-id="1*EKy6WDVn4uPbO7OpQNJRcg.png" data-width="772" data-height="663" src="https://cdn-images-1.medium.com/max/800/1*EKy6WDVn4uPbO7OpQNJRcg.png"></div><figcaption class="imageCaption">The area of interest partitioned into a grid of ‘patches’</figcaption></figure><h4 name="dd81" id="dd81" class="graf graf--h4 graf-after--figure">2. Loading the image data from disk</h4><p name="ef41" id="ef41" class="graf graf--p graf-after--h4">The image data for the competition was supplied in JPEG2000 format in the standard <a href="https://sentinel.esa.int/web/sentinel/user-guides/sentinel-2-msi/data-formats" data-href="https://sentinel.esa.int/web/sentinel/user-guides/sentinel-2-msi/data-formats" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">Sentinel2 folder structure</a> illustrated below:</p><figure name="33d2" id="33d2" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 256px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 36.5%;"></div><img class="graf-image" data-image-id="1*CbyCQhd1R031kOXOMPkI7Q.jpeg" data-width="1100" data-height="402" src="https://cdn-images-1.medium.com/max/800/1*CbyCQhd1R031kOXOMPkI7Q.jpeg"></div><figcaption class="imageCaption">Sentinel2 folder structure</figcaption></figure><p name="d4ce" id="d4ce" class="graf graf--p graf-after--figure">The eo-learn library has many useful predefined tasks for loading imagery from Sentinel Hub, manipulating imagery and generating features. At the time of writing it did not have a task to load imagery from disk in the format specified above. Nevertheless, defining my own <code class="markup--code markup--p-code">EOTask</code> class to do this proved simple enough. <code class="markup--code markup--p-code">EOTask</code> classes need an <code class="markup--code markup--p-code">execute()</code> method that optionally takes an <code class="markup--code markup--p-code">EOPatch</code> object as an argument.</p><p name="4d64" id="4d64" class="graf graf--p graf-after--p"><code class="markup--code markup--p-code">EOPatch</code> objects are essentially just collections of <code class="markup--code markup--p-code">numpy</code> arrays along with metadata. The <code class="markup--code markup--p-code">EOPatch</code> objects loaded by my own custom <code class="markup--code markup--p-code">EOTask</code> looked something like this:</p><pre name="6a84" id="6a84" class="graf graf--pre graf-after--p"><code class="markup--code markup--pre-code">data: {<br>    BANDS: numpy.ndarray(shape=(11, 1345, 1329, 13), dtype=float64)<br>  }<br>  mask: {}<br>  mask_timeless: {}<br>  scalar_timeless: {}<br>  label_timeless: {}<br>  vector_timeless: {}<br>  meta_info: {<br>    service_type: &#39;wcs&#39;<br>    size_x: &#39;10m&#39;<br>    size_y: &#39;10m&#39;<br>  }<br>  bbox: BBox(((535329.7703788084, 6846758.109461494), (548617.0052632861, 6860214.913734847)), crs=EPSG:32734)<br>  timestamp: [datetime.datetime(2017, 1, 1, 8, 23, 32), ..., datetime.datetime(2017, 8, 19, 8, 20, 11)], length=11<br>)</code></pre><p name="ca53" id="ca53" class="graf graf--p graf-after--pre">We can visualise the patches by using bands 4, 3 and 2 (red, green and blue) to generate a colour image for each patch:</p><figure name="7f4a" id="7f4a" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 576px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 82.19999999999999%;"></div><img class="graf-image" data-image-id="1*zaoHmpWKwIyOb339LK-_hg.png" data-width="1351" data-height="1111" src="https://cdn-images-1.medium.com/max/800/1*zaoHmpWKwIyOb339LK-_hg.png"></div><figcaption class="imageCaption">Colour images of the 12 patches made with the red, green and blue bands</figcaption></figure><h4 name="0b4d" id="0b4d" class="graf graf--h4 graf-after--figure">3. Masking out clouds</h4><p name="7eb0" id="7eb0" class="graf graf--p graf-after--h4">In the bottom right corner of the above image there is some cloud cover. The eo-learn library provides a pre-trained pixel-level cloud detector model. This functionality is available through the <code class="markup--code markup--p-code">S2PixelCloudDetector</code><em class="markup--em markup--p-em"> </em>and<em class="markup--em markup--p-em"> </em>the<code class="markup--code markup--p-code">AddCloudMaskTask</code> classes.</p><p name="c00b" id="c00b" class="graf graf--p graf-after--p">The <code class="markup--code markup--p-code">S2PixelCloudDetector</code> comes from a separate library <a href="https://github.com/sentinel-hub/sentinel2-cloud-detector" data-href="https://github.com/sentinel-hub/sentinel2-cloud-detector" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">sentinel2-cloud-detector</a> and uses all 13 bands of the Sentinel2 imagery to make its predictions. By setting a probability threshold the cloud probability predictions can be turned into a cloud mask.</p></div><div class="section-inner sectionLayout--outsetColumn"><figure name="1ca4" id="1ca4" class="graf graf--figure graf--layoutOutsetCenter graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 1032px; max-height: 313px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 30.3%;"></div><img class="graf-image" data-image-id="1*ui_wN1SrPy5jV1Y6KY_zeQ.png" data-width="1131" data-height="343" src="https://cdn-images-1.medium.com/max/1200/1*ui_wN1SrPy5jV1Y6KY_zeQ.png"></div><figcaption class="imageCaption">Colour image with clouds, cloud probabilities and resulting cloud mask</figcaption></figure></div><div class="section-inner sectionLayout--insetColumn"><p name="0dcb" id="0dcb" class="graf graf--p graf-after--figure">I used this cloud detection functionality to add a cloud mask to my data.</p><h4 name="b1d0" id="b1d0" class="graf graf--h4 graf-after--p">4. Resampling through time</h4><p name="6845" id="6845" class="graf graf--p graf-after--h4">Cutting out clouds leaves gaps in the data for the areas with cloud cover in each time slice. One possible approach to filling these gaps is to interpolate between preceding and subsequent time slices.</p><p name="88ae" id="88ae" class="graf graf--p graf-after--p">There’s already a <code class="markup--code markup--p-code">LinearInterpolation</code> <code class="markup--code markup--p-code">EOTask</code> defined for this purpose. The class requires that you specify which bands to interpolate and an interval to resample on. I decided to average out my data to approximately one time slice per month, which reduced my time dimension from 11 time points to 8.</p><p name="dd16" id="dd16" class="graf graf--p graf-after--p">Additionally, to deal with any gaps at the start or end of the time period I used a <code class="markup--code markup--p-code">ValueFilloutTask</code> for simple extrapolation by copying values from preceding or succeeding time points as necessary.</p><h4 name="89fc" id="89fc" class="graf graf--h4 graf-after--p">5. Adding NDVI</h4><p name="9420" id="9420" class="graf graf--p graf-after--h4"><em class="markup--em markup--p-em">Normalized difference vegetation index </em>(NDVI) is a simple indicator of the presence of plant life in satellite imagery. The index is calculated using the red and near infra-red (NIR) bands.</p><p name="98ff" id="98ff" class="graf graf--p graf-after--p">NDVI = (NIR - Red)/(NIR + Red)</p><p name="2573" id="2573" class="graf graf--p graf-after--p">The Wikipedia article on NDVI has a <a href="https://en.wikipedia.org/wiki/Normalized_difference_vegetation_index#Rationale" data-href="https://en.wikipedia.org/wiki/Normalized_difference_vegetation_index#Rationale" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">nice explanation</a> for the rationale behind this indicator. The essential idea is that plant matter absorbs much of visible red spectrum light while it reflects near infrared light which it cannot use for photosynthesis, NDVI captures this difference in reflectance in a ratio.</p><p name="bb90" id="bb90" class="graf graf--p graf-after--p">Conveniently eo-learn provides an <code class="markup--code markup--p-code">NormalizedDifferenceIndex</code> task which allowed me to easily compute and add NDVI for each of the patches.</p><p name="8da7" id="8da7" class="graf graf--p graf-after--p">NDVI evolves differently through time for different crops. Different crops are planted and harvested at different times and grow at different rates. The animation below shows how NDVI evolves differently for adjacent fields.</p><figure name="0345" id="0345" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 774px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 110.60000000000001%;"></div><img class="graf-image" data-image-id="1*xBky4ockxbzajTUyh8Evug.gif" data-width="705" data-height="780" src="https://cdn-images-1.medium.com/max/800/1*xBky4ockxbzajTUyh8Evug.gif"></div><figcaption class="imageCaption">NDVI through time (in March you can see artefacts that result from the cloud masking and interpolation)</figcaption></figure><h4 name="d012" id="d012" class="graf graf--h4 graf-after--figure">6. Adding the target masks</h4><p name="cb84" id="cb84" class="graf graf--p graf-after--h4">In order to treat the crop identification challenge as a semantic segmentation task I needed to create target masks for our imagery. The <code class="markup--code markup--p-code">VectorToRaster</code> task in eo-learn takes vector geometries and creates a rasterised layer. I used this task to add a raster layer indicating the crop types. I also added a layer with the field identifiers for use in inference.</p><figure name="e684" id="e684" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 717px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 102.4%;"></div><img class="graf-image" data-image-id="1*tEBvN5ikY-mKN31OvyINdg.png" data-width="1042" data-height="1067" src="https://cdn-images-1.medium.com/max/800/1*tEBvN5ikY-mKN31OvyINdg.png"></div><figcaption class="imageCaption">Crop type raster layer for patch number 6</figcaption></figure><h4 name="f55f" id="f55f" class="graf graf--h4 graf-after--figure">Creating a workflow and executing it</h4><p name="661c" id="661c" class="graf graf--p graf-after--h4">To run each of the above preprocessing steps I put all the tasks into a workflow. In general, an eo-learn workflow can be any acyclic directed graph with <code class="markup--code markup--p-code">EOTask</code> objects at each node. I just used a linear workflow which looked something like:</p><pre name="ed62" id="ed62" class="graf graf--pre graf-after--p">LinearWorkflow(<br> add_data,           # load the data<br> add_clm,            # create cloud mask<br> ndvi,               # compute ndvi<br> norm,               # compute the euclidean norm of the bands<br> concatenate         # add the ndvi and norm to the bands<br> linear_interp,      # linear interpolation<br> fill_extrapolate,   # extrapolation<br> target_raster,      # add target masks<br> field_id_raster,    # add field identifiers<br> save                # save the data back to disk<br>)</pre><p name="36e5" id="36e5" class="graf graf--p graf-after--pre">To execute this workflow I created execution arguments for each patch and then used an <code class="markup--code markup--p-code">EOExecutor</code> to run the entire workflow on all of the patches in a distributed fashion.</p><pre name="e1ee" id="e1ee" class="graf graf--pre graf-after--p">execution_args = []<br>for patch_idx in range(12):<br>    execution_args.append({<br>        load: {&#39;eopatch_folder&#39;: f&#39;eopatch_{patch_idx}&#39;},<br>        save: {&#39;eopatch_folder&#39;: f&#39;eopatch_{patch_idx}&#39;}<br>    })<br>    <br>executor = EOExecutor(workflow, execution_args, save_logs=True)<br>executor.run(workers=6, multiprocess=False)</pre><h3 name="7e6a" id="7e6a" class="graf graf--h3 graf-after--pre">Building a prediction model</h3><h4 name="9ca4" id="9ca4" class="graf graf--h4 graf-after--h3">Semantic segmentation</h4><p name="fa83" id="fa83" class="graf graf--p graf-after--h4">Semantic segmentation is the process of assigning a class label to each pixel of an image. By reframing the crop identification problem in this challenge as a semantic segmentation task I could take advantage of the information in the local spatial context of each field, as I show below, it also allowed me to generate more training data by repeated sampling.</p><h4 name="3441" id="3441" class="graf graf--h4 graf-after--p">1. Generating the training set (<a href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Create%20Unet%20Features.ipynb" data-href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Create%20Unet%20Features.ipynb" class="markup--anchor markup--h4-anchor" rel="noopener" target="_blank">notebook</a>)</h4><p name="a77d" id="a77d" class="graf graf--p graf-after--h4">From my 12 patches I randomly sampled 64 x 64 pixel ‘patchlets’ to train my model. I kept the patch size small as the fields themselves are relatively small and the provided Sentinel2 imagery has a maximum spatial resolution of 10m . This means a square field 1 hectare in size (10,000m²) is appears in the imagery as an area of 32 x 32 pixels.</p><p name="c708" id="c708" class="graf graf--p graf-after--p">I sampled the patchlets in a manner that ensured that each patchlet contained at least a part of a training field. For each patchlet I saved two pickle files, one containing the input imagery and the other the raster layer with the crop types.</p><p name="e344" id="e344" class="graf graf--p graf-after--p">For the input imagery I chose to include six channels, the three visible bands (red, green and blue), near infra-red and the calculated NDVI and euclidean norm. When I resampled the imagery by interpolating through time I ended up with eight different time points. In order to get a rank three tensor, I simply stacked the six channels at each of the eight time points to get a 48-channel image.</p></div><div class="section-inner sectionLayout--outsetColumn"><figure name="8304" id="8304" class="graf graf--figure graf--layoutOutsetCenter graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 1032px; max-height: 338px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 32.7%;"></div><img class="graf-image" data-image-id="1*LN8c786HRp40pyXxRvCp6w.png" data-width="2114" data-height="692" src="https://cdn-images-1.medium.com/max/1200/1*LN8c786HRp40pyXxRvCp6w.png"></div><figcaption class="imageCaption">NDVI and visible images at a single time point along with the corresponding target crop types for nine randomly sampled 64x64 training ‘patchlets’</figcaption></figure></div><div class="section-inner sectionLayout--insetColumn"><h4 name="a559" id="a559" class="graf graf--h4 graf-after--figure">2. Data augmentation</h4><p name="5ffc" id="5ffc" class="graf graf--p graf-after--h4">The relatively small dataset available in the competition and the large number of parameters in my chosen network architecture meant that I needed to be particularly careful of overfitting. To avoid this I use image augmentation as well as <a href="https://arxiv.org/abs/1710.09412" data-href="https://arxiv.org/abs/1710.09412" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">mixup</a>.</p><p name="06e0" id="06e0" class="graf graf--p graf-after--p">The fastai library provides an array of image augmentation techniques. I used:</p><ul class="postList"><li name="3be1" id="3be1" class="graf graf--li graf-after--p">vertical flips</li><li name="c85a" id="c85a" class="graf graf--li graf-after--li">horizontal flips</li><li name="dd7e" id="dd7e" class="graf graf--li graf-after--li">rotation</li><li name="8140" id="8140" class="graf graf--li graf-after--li">zoom</li><li name="9734" id="9734" class="graf graf--li graf-after--li">warping</li><li name="a06b" id="a06b" class="graf graf--li graf-after--li">and cutout</li></ul><figure name="de7d" id="de7d" class="graf graf--figure graf-after--li"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 701px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 100.1%;"></div><img class="graf-image" data-image-id="1*eqqf_wXMHCthUdjIIyGElQ.png" data-width="1430" data-height="1432" src="https://cdn-images-1.medium.com/max/800/1*eqqf_wXMHCthUdjIIyGElQ.png"></div><figcaption class="imageCaption">A batch of training images with image augmentations applied</figcaption></figure><h4 name="0772" id="0772" class="graf graf--h4 graf-after--figure">3. Creating a fastai U-Net model (<a href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Train%20Unet%20Model.ipynb" data-href="https://github.com/simongrest/farm-pin-crop-detection-challenge/blob/master/notebooks/Train%20Unet%20Model.ipynb" class="markup--anchor markup--h4-anchor" rel="noopener" target="_blank">notebook</a>)</h4><p name="0cc1" id="0cc1" class="graf graf--p graf-after--h4">The <a href="https://docs.fast.ai/" data-href="https://docs.fast.ai/" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">fastai</a> library provides for semantic segmentation by allowing the user to dynamically build a U-Net from an existing convolutional network encoder. I chose a ResNet50 pre-trained on <a href="www.image-net.org" data-href="www.image-net.org" class="markup--anchor markup--p-anchor" target="_blank">ImageNet</a> as my encoder network. To deal with the shape of my input tensors I replaced the first convolutional layer of the ResNet50 network that takes 3 channels with one that takes 48 channels instead.</p><p name="7ab3" id="7ab3" class="graf graf--p graf-after--p">I won’t attempt to explain U-Nets or residual neural networks here as there are many good explanations available already. For example here’s a <a href="https://towardsdatascience.com/understanding-semantic-segmentation-with-unet-6be4f42d4b47" data-href="https://towardsdatascience.com/understanding-semantic-segmentation-with-unet-6be4f42d4b47" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">post</a> explaining U-Nets and here’s <a href="https://towardsdatascience.com/introduction-to-resnets-c0a830a288a4" data-href="https://towardsdatascience.com/introduction-to-resnets-c0a830a288a4" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">another</a> explaining ResNets.</p><p name="b7aa" id="b7aa" class="graf graf--p graf-after--p">I created <code class="markup--code markup--p-code">SegmentationPklList</code> and classes<code class="markup--code markup--p-code">SegmentationPklLabelList</code> to implement functionality to load pickle file ‘images’ so that my data worked with the fastai’s data block API.</p><p name="4ee1" id="4ee1" class="graf graf--p graf-after--p">The fastai <code class="markup--code markup--p-code">MixUpCallback</code> and <code class="markup--code markup--p-code">MixUpLoss</code> also needed some minor tweaking to work with semantic segmentation.</p><h4 name="a1d7" id="a1d7" class="graf graf--h4 graf-after--p">4. Loss function</h4><p name="239c" id="239c" class="graf graf--p graf-after--h4">I used a modified<code class="markup--code markup--p-code">CrossEntropyFlat</code> loss function to score my model. I instantiated it as:</p><p name="0fd9" id="0fd9" class="graf graf--p graf-after--p"><code class="markup--code markup--p-code">CrossEntropyFlat(axis=1, weight=inv_prop, ignore_index=0)</code></p><p name="7ed7" id="7ed7" class="graf graf--p graf-after--p">The occurrence of the different crop types in the training set was imbalanced, certain of the crop types only occurring a handful of times. I weighted my loss function in proportion with the inverse frequency of each crop type by using the <code class="markup--code markup--p-code">weight</code> parameter of the loss constructor.</p><p name="ed82" id="ed82" class="graf graf--p graf-after--p">Much of the area of the training images did not have a crop type, either there was no field in that region, or it if there was a field it was not part of the training set. I ignored predictions where there is no crop type label by using the <code class="markup--code markup--p-code">ignore_index</code> parameter of the loss constructor.</p><h4 name="33aa" id="33aa" class="graf graf--h4 graf-after--p">5. Training</h4><p name="80e9" id="80e9" class="graf graf--p graf-after--h4">One of the biggest advantages that the fastai library offers is a flexible training loop along with great out of the box support for controlling training parameters through techniques such as the <a href="https://arxiv.org/pdf/1803.09820.pdf" data-href="https://arxiv.org/pdf/1803.09820.pdf" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">one cycle training policy</a>. I trained my U-Net for five epochs using the <code class="markup--code markup--p-code">fit_one_cycle</code> function keeping the pre-trained encoder parameters frozen, and then for a further ten epochs allowing the encoder weights to be updated.</p><p name="8f40" id="8f40" class="graf graf--p graf-after--p">During training the loss on the validation set decreased consistently and my custom pixel accuracy metric increased fairly steadily.</p><figure name="02e2" id="02e2" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 700px; max-height: 519px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 74.2%;"></div><img class="graf-image" data-image-id="1*DZ8dqPjE-ZVAx-zOxqCHDg.png" data-width="1200" data-height="890" src="https://cdn-images-1.medium.com/max/800/1*DZ8dqPjE-ZVAx-zOxqCHDg.png"></div><figcaption class="imageCaption">Training results from 5 frozen and 10 unfrozen epochs</figcaption></figure><p name="6cf2" id="6cf2" class="graf graf--p graf-after--figure">Comparing the predicted pixel masks to the target masks for examples in the validation set seemed to indicate that the network was working reasonably but that that there were examples of poor performance on the minority classes and fields with non-standard shapes.</p></div><div class="section-inner sectionLayout--outsetColumn"><figure name="be64" id="be64" class="graf graf--figure graf--layoutOutsetCenter graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 1032px; max-height: 752px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 72.89999999999999%;"></div><img class="graf-image" data-image-id="1*kb2jtZTMDkLmQdB4ygvOYQ.png" data-width="1754" data-height="1278" src="https://cdn-images-1.medium.com/max/1200/1*kb2jtZTMDkLmQdB4ygvOYQ.png"></div></figure><figure name="9a52" id="9a52" class="graf graf--figure graf--layoutOutsetCenter graf-after--figure"><div class="aspectRatioPlaceholder is-locked" style="max-width: 1032px; max-height: 762px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 73.9%;"></div><img class="graf-image" data-image-id="1*7e3odw0qLMbPWTsnj2XGvw.png" data-width="1752" data-height="1294" src="https://cdn-images-1.medium.com/max/1200/1*7e3odw0qLMbPWTsnj2XGvw.png"></div><figcaption class="imageCaption">More predictions on examples from the validation set</figcaption></figure></div><div class="section-inner sectionLayout--insetColumn"><h3 name="05b7" id="05b7" class="graf graf--h3 graf-after--figure">Results and areas for improvement</h3><p name="964b" id="964b" class="graf graf--p graf-after--h3">To do inference on the test set, I divided up each patch into a grid of 64x64 ‘patchlets’ and saved the pickle files for each patchlet. I made predictions for the entire test set and grouped the result by the<code class="markup--code markup--p-code">Field_Id</code>. Predictions for each pixel consisted of the ten final activations from the U-Net. I took the median activation value for each class, and then applied a softmax function to get a single probability per crop for every <code class="markup--code markup--p-code">Field_Id</code> in the test set.</p><figure name="6162" id="6162" class="graf graf--figure graf-after--p"><div class="aspectRatioPlaceholder is-locked" style="max-width: 650px; max-height: 704px;"><div class="aspectRatioPlaceholder-fill" style="padding-bottom: 108.3%;"></div><img class="graf-image" data-image-id="1*mcKa5KTQAN_tGEfI-opnEw.png" data-width="650" data-height="704" src="https://cdn-images-1.medium.com/max/800/1*mcKa5KTQAN_tGEfI-opnEw.png"></div><figcaption class="imageCaption">sample grid of ‘patchlets’ for inference — colouring by Field_Id clearly shows the data leak</figcaption></figure><h4 name="7cce" id="7cce" class="graf graf--h4 graf-after--figure">Taking advantage of temporal patterns</h4><p name="259a" id="259a" class="graf graf--p graf-after--h4">Reflecting on my approach, I think that the area where the most improvement could have been made was in the treatment of the time dimension. My naive approach of stacking all the timepoints in 48 channels does not allow my model to properly learn from patterns in the imagery through time. I would have liked to explore using recurrent networks to learn these temporal patterns.</p><p name="a74d" id="a74d" class="graf graf--p graf-after--p">The team behind eo-learn have themselves have proposed using a <em class="markup--em markup--p-em">Temporal Fully-Convolutional Network</em> (TFCN) for this: <a href="https://sentinel-hub.com/sites/default/lps_2019_eolearn_TFCN.pdf" data-href="https://sentinel-hub.com/sites/default/lps_2019_eolearn_TFCN.pdf" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">https://sentinel-hub.com/sites/default/lps_2019_eolearn_TFCN.pdf</a>. TFCNs take rank 4 tensors as inputs and use 3D convolutions to capture patterns in space and time simultaneously.</p><h4 name="41b5" id="41b5" class="graf graf--h4 graf-after--p">Building latent representations using unsupervised learning</h4><p name="a89c" id="a89c" class="graf graf--p graf-after--h4">If the competition had allowed for the use of external data, it would have been interesting to explore the <em class="markup--em markup--p-em">Tile2Vec</em> technique described in this paper <a href="https://arxiv.org/abs/1805.02855" data-href="https://arxiv.org/abs/1805.02855" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank">https://arxiv.org/abs/1805.02855</a>. The idea here is to generate latent vector representations of areas from satellite imagery by setting up an unsupervised learning task using a triplet loss.</p><p name="6c7f" id="6c7f" class="graf graf--p graf-after--p">I am very curious to learn what approaches the other competitors employed.</p><h3 name="f327" id="f327" class="graf graf--h3 graf-after--p"><em class="markup--em markup--h3-em">Thanks</em></h3><p name="7cd6" id="7cd6" class="graf graf--p graf-after--h3 graf--trailing"><em class="markup--em markup--p-em">I’d like to thank the team at </em><a href="http://zindi.africa" data-href="http://zindi.africa" class="markup--anchor markup--p-anchor" rel="noopener" target="_blank"><em class="markup--em markup--p-em">Zindi</em></a><em class="markup--em markup--p-em"> for putting together such an interesting challenge.
